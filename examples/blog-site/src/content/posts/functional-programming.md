## 什么是函数式编程

函数式编程（FP）是一种以函数为核心构建块的编程范式。它强调使用纯函数、避免可变状态和副作用。与面向对象编程关注"事物是什么"不同，函数式编程关注"数据如何变换"。虽然 Haskell 等纯函数式语言并非主流，但 FP 的核心思想已深刻影响了 JavaScript、Python、Rust 等通用语言的设计。

## 核心概念

### 纯函数

纯函数对相同的输入始终返回相同的输出，且不产生副作用（不修改外部状态、不进行 I/O 操作）。纯函数天然可测试、可缓存、可并行执行。判断一个函数是否纯净，只需问一个问题：把它的调用替换为返回值，程序行为是否不变？

### 不可变性

数据一旦创建就不能被修改。需要"修改"时，创建一个包含变更的新副本。这消除了共享可变状态带来的一系列问题：竞态条件、意外的远程修改、难以追踪的 Bug。JavaScript 中可以使用 `Object.freeze`、展开运算符或 Immer 库实现不可变数据操作。

### 高阶函数

能接受函数作为参数或返回函数的函数称为高阶函数。`map`、`filter`、`reduce` 是最常用的高阶函数，它们将循环逻辑抽象为声明式操作，使代码意图更加清晰。

### 函数组合

将多个简单函数组合为一个复杂函数，就像搭积木一样构建程序。Unix 管道哲学（每个工具做好一件事，通过管道串联）就是函数组合的经典体现。`compose(f, g)` 等价于 `x => f(g(x))`，数据像在流水线上一样依次经过每个函数的加工。

## 实际应用

### React 中的函数式思想

React 的函数组件和 Hooks 深受 FP 影响。组件是从 Props 到 UI 的纯函数映射，`useState` 通过替换而非修改状态来触发重新渲染，`useMemo` 体现了引用透明带来的缓存优势。

### 数据处理管道

函数式编程非常适合数据处理场景。将复杂的数据转换分解为一系列小的纯函数，通过管道串联起来。每一步都可以独立测试和复用，整个管道的行为也容易推理和调试。

## 务实地拥抱 FP

不必追求纯粹的函数式编程。将 FP 的核心思想——纯函数、不可变性、函数组合——融入日常编码实践，就能写出更可预测、更易测试的代码。在需要副作用的地方（网络请求、数据库操作）将其推到系统边界，保持核心业务逻辑的纯净性。
